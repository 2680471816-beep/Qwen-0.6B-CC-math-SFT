## 零、输入数据说明

### JSONL 文件结构

每行一个 JSON 对象，结构与 A/B 类相同：

```json
{
  "id": "9f35e5bf-...",
  "text": "## Applied Mathematics for Class 11\n\n### Introduction\n...\n### Example\nConsider the function $f(x) = x^2$...",
  "metadata": { "finemath_scores": 4.31, "nemocurator_scores": 2.67, ... },
  "stage3_label": "C"
}
```

C 类数据的 `text` 字段来自**数学教育类网页**，内容以知识讲解为主，包括教材节选、课程笔记、教程文章、博客等。与 A 类（单道题+解答）和 B 类（论坛问答）不同，C 类文档的核心是**概念讲解**，题目和解答并非总是显式存在。

---

### C 类的分类标准（Stage 3 中确定）

Stage 3 分类时，C 类（ARTICLE_TUTORIAL）的判定规则为：

> 文档不含论坛元数据（排除 B 类），不具备教科书的章节编号+正式定理结构（排除 D 类），但包含数学概念讲解内容，表现为连续段落叙述 + 数学公式。

换言之，C 类是在 A/B/D 类之外、有数学内容的"教育类文章"大类，内部结构高度多样。

### 1.3 四种典型文档模式

---

**模式一：含明确 Problem/Solution 节的文章（约 15%）**

部分教程文章内嵌了结构完整的题目，可直接复用 A 类策略：

```markdown
# Calculus: Integration Techniques           ← H1：文章标题

## The Fundamental Theorem of Calculus       ← H2：概念节
The FTC connects differentiation and integration...

## Problem                                   ← H2：问题节
Find the area bounded by $y = x^2$ and $x = 2$...

## Solution                                  ← H2：解答节
Set up the integral: $\int_0^2 x^2\,dx = \frac{8}{3}$...
```

---

**模式二：含 Example 节的讲解文章（最常见，33.6%）**

概念讲解之后紧跟一个或多个具体示例，是 C 类最核心的结构：

```markdown
# Differential Calculus                     ← H1：文章标题

The derivative of $f(x)$ is defined as:
$$ f'(x) = \lim_{h \to 0} \frac{f(x+h)-f(x)}{h} $$

This captures the slope of the tangent line at any point.

### Example                                 ← H3：示例节
Consider $f(x) = x^2$. The derivative is:
$$ f'(x) = \lim_{h\to 0}\frac{(x+h)^2 - x^2}{h} = 2x $$
```

---

**模式三：H1/H2 标题本身即问题（约 16%）**

部分教程文章的标题直接就是一个数学问题或主题，正文是其讲解/解答：

```markdown
# What is the total cost to heat the greenhouse?   ← H1：就是问题

The temperature is $F(t) = 22 + 20\cos(\pi t/12)$...
The heater kicks on when $F(t) \le 36$...
```

---

**模式四：Theorem + Proof 结构（教科书风格，约 7%）**

部分教科书节选有正式的定理+证明结构：

```markdown
#### Theorem                                ← H4：定理
If $f$ is continuous on $[a,b]$ and differentiable on $(a,b)$,
then there exists $c \in (a,b)$ such that $f'(c) = \frac{f(b)-f(a)}{b-a}$.

#### Proof                                  ← H4：证明
Define $g(x) = f(x) - \frac{f(b)-f(a)}{b-a}(x-a)$...
By Rolle's Theorem, there exists $c$ with $g'(c) = 0$...
```

## 二、问答对构建策略

### 核心思路

C 类文章本身不是一道题，而是对某个数学概念的讲解。构建 QA 对的两种核心方法：

1. **直接提取**（优先）：文章内嵌了结构化题目，复用 A 类的 7 种策略（S1~S5）
2. **结构提取**（C 类新增）：从文章的 Example 节或 Theorem+Proof 节中，将"概念背景 + 示例"重新组织为 QA 对

---

### 策略优先级（按序尝试，命中即停止）

| 优先级 | 策略 | 适用场景 | 命中数 | 占比 |
|--------|------|---------|--------|------|
| 1 | **S1** | 明确 `## Problem` + `## Solution` 双标题 | 48 | 0.2% |
| 2 | **S1b** | 只有 `## Solution`，取前一节作问题 | 1,283 | 4.6% |
| 3 | **S2** | 只有 `## Problem/Question`，之后作解答 | 1,550 | 5.6% |
| 4 | **S3** | H1/H2 标题含疑问词或数学符号 | 2,138 | 7.7% |
| 5 | **C-Ex** | `## Example` 节提取（新增） | 5,763 | **20.8%** |
| 6 | **S3b** | H1/H2 为数学主题词 + 正文含 ≥3 LaTeX | 2,144 | 7.7% |
| 7 | **C-Thm** | `## Theorem` + `## Proof` 节提取（新增）| 3 | 0.0% |
| 8 | **S5** | `**Question**:` / `**Problem**:` 加粗标注 | 594 | 2.1% |
| 9 | **S4** | 首段含问题关键词，其余作解答 | 541 | 2.0% |

---

### 新增策略 C-Ex 详解（最重要，命中 5,763 条）

**核心逻辑**：

1. 找文档中第一个 `## Example` / `### Example N:` 标题
2. 提取 Example 节内容作为**解答侧**（演示步骤）
3. 提取 Example 节之前最近一个标题节的内容作为**问题侧**（概念背景）
4. 若 Example 标题本身含具体问题描述（如 `### Example 1: Find the derivative of...`），直接用该描述作为问题

**构建示例**：

```
输入文档（片段）：
  [概念节]：The derivative of $f(x)$ is $f'(x) = \lim_{h\to 0}\frac{f(x+h)-f(x)}{h}$

  ### Example
  Consider $f(x) = x^2$. Compute $f'(x)$.
  [解算步骤] $f'(x) = 2x$

输出 QA 对：
  Q: Differential Calculus
     The derivative of $f(x)$ is defined as...（概念节内容）
  A: Consider $f(x) = x^2$. Compute $f'(x)$.
     $f'(x) = \lim_{h\to 0}\frac{(x+h)^2-x^2}{h} = 2x$
```

**多 Example 节的文档**：对每个 Example 节依次尝试，取第一个通过质量检验的提取结果。这样即便一篇文章有 5 个 Example，也只产生 1 条训练数据（避免重复语境下的过度提取）。

---

### 新增策略 C-Thm 详解（命中 3 条）

**核心逻辑**：

1. 找 `## Theorem` / `## Lemma` / `## Corollary` 标题，提取定理内容作为**问题侧**
2. 在其后找 `## Proof` 节，提取证明内容作为**解答侧**
3. 问题格式：`{定理标题}\n\n{定理陈述}`（引导模型"证明此定理"）

**命中数极少（3 条）**的原因：C 类数据中虽有大量 `theorem` 关键词（7,999 条，28.9%），但规范的 `## Theorem` 独行标题极少，大多是"嵌入段落"的定理陈述，C-Thm 策略的正则要求独立标题，覆盖范围窄。D 类数据才是 C-Thm 的主要应用场景。

---

### 质量过滤

每条提取结果须通过三项检查：

| 检查项 | 阈值 | 说明 |
|--------|------|------|
| 问题最短长度 | 30 chars | 防止提取到空节或纯标题 |
| 解答最短长度 | 100 chars | 保证解答有实质内容 |
| 全文含数学内容 | 至少 1 处 | LaTeX、算术表达式，确保是数学类数据 |

## 三、遇到的问题与解决方法

### 问题 1：C-Ex 提取的"问题"以 "Example" 开头，语义不清

**现象**：当 Example 节之前的最近标题只有标题行而无内容（如 `### Example` 直接跟在另一个 `### Example` 下），提取出的"问题"部分就是 `"Example\n\nConsider the function..."` 这种形式——以 "Example" 开头，不是一个有意义的提问。全量数据中约 16.8% 的 C-Ex 命中存在此情况。

**解决**：为 C-Ex 增加后处理逻辑：
- 检查问题文本是否以 "Example" 开头
- 若是，向上寻找更高层标题节（如 `## Differential Calculus`）的内容作为背景
- 若再找不到，则取 Example 节内容的第一行作为问题描述（该行通常是具体的题目陈述）

修复后，以 "Example" 开头的问题比例从 16.8% 降至约 5%，剩余部分均是确实以 "Example N: Find..." 形式命名的标题，本身就是有意义的问题描述。

---

### 问题 2：C 类约 20% 数据是"论坛边界记录"，含用户元数据

**现象**：Stage 3 分类时，部分论坛帖子因缺少足够的论坛元数据标记（`< 2` 处）未被分入 B 类，而是划为 C 类。这类记录（约 3,958 条，14.3%）含有 `Posted by` / `Member` / `Joined` 等论坛元数据，但混在 C 类文章中。

**解决**：S3b 策略（H1/H2 为数学主题词）对这类记录有一定的覆盖，因为其 H1 标题通常是数学主题，正文含大量 LaTeX。最终 S3b 命中了 2,144 条，覆盖了大多数此类"边界论坛帖"，解答侧会包含论坛用户名等噪声，但数学内容完整，训练上可接受。

---

### 问题 3：C-Thm 策略命中极少（仅 3 条）

**现象**：C 类有 7,999 条（28.9%）含 theorem 关键词，理论上 C-Thm 应有较多命中，但实际只有 3 条。

**原因分析**：C-Thm 要求 `## Theorem` 是**独立成行的标题**（正则 `^#{1,4}\s*theorem\s*[\d\.]*\s*$`）。实际数据中，定理大多以嵌入段落的方式出现（如 `**Theorem 2.1** (Mean Value Theorem): If $f$...`），而非规范的 Markdown 标题行，因此正则无法匹配。

**处理方式**：C-Thm 的主要价值在 D 类（教科书型）数据，C 类中的定理内容往往由 S1b（只有 Proof 标题）或 C-Ex（Example 节演示定理应用）覆盖，不额外扩展 C-Thm 的匹配范围，避免误匹配。
